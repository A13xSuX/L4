## Описание

Этот проект демонстрирует паттерн **or-channel** на Go.

Идея паттерна в том, чтобы объединить несколько каналов в один общий канал завершения. Как только **хотя бы один** из входных каналов срабатывает или закрывается, общий канал тоже завершается. Это удобно в конкурентных программах, когда нужно ждать сразу несколько источников сигнала и продолжать выполнение после первого из них.

В проекте реализована функция `Or(channels ...<-chan interface{}) <-chan interface{}`, которая:

- принимает произвольное количество каналов только для чтения;
- возвращает один объединённый канал;
- закрывает его, когда срабатывает любой из входных каналов.

#### Логика работы

1. Если каналов нет, функция возвращает `nil`.
2. Если передан один канал, функция просто возвращает его.
3. Если каналов два, запускается `select`, который ждёт первый завершившийся канал.
4. Если каналов больше двух, используется рекурсивное объединение:
   - функция следит за первыми тремя каналами;
   - оставшиеся объединяются рекурсивным вызовом `Or(...)`;
   - дополнительно используется `orDone`, чтобы корректно завершать вложенные ожидания.
